# Prompt para Replit AI — Fase 3: Convertir la app de Pilates en un SaaS listo para vender

Convierte mi app actual de gestión para estudios de pilates en una plataforma SaaS multi-estudio, profesional y lista para producción.

La app ya tiene:

* Login
* Dashboard
* Clientes
* Clases
* Reservas
* Instructores
* Asistencia
* Membresías
* Pagos con Stripe
* Recordatorios por WhatsApp
* Dashboard con gráficas

Stack actual:

* Frontend: React + Next.js + Tailwind + Framer Motion

* TypeScript estricto

# 1. Multi-estudio / Multi-tenant

Crear tabla PostgreSQL:

```sql
studios (
  id,
  name,
  slug,
  logoUrl,
  primaryColor,
  secondaryColor,
  phone,
  email,
  address,
  createdAt
)
```

Actualizar tablas:

* users
* clients
* instructors
* classes
* bookings
* memberships
* payments

Agregar `studioId` como foreign key.

Reglas:

* Cada usuario pertenece a un estudio.
* Ningún estudio puede ver información de otro.
* Filtrar automáticamente todas las consultas por `studioId`.
* El login debe identificar a qué estudio pertenece el usuario.

# 2. Roles y permisos

Agregar roles:

* ADMIN
* RECEPTIONIST
* INSTRUCTOR

Actualizar tabla `users`:

```sql
role ENUM('ADMIN','RECEPTIONIST','INSTRUCTOR')
```

Permisos:

* ADMIN:

  * acceso total
* RECEPTIONIST:

  * puede crear clientes, reservas y cobrar
  * no puede borrar pagos ni modificar configuración
* INSTRUCTOR:

  * solo puede ver sus clases, alumnos y asistencia

Backend:

* Middleware `requireRole()`
* Proteger rutas según rol

Frontend:

* Mostrar/ocultar secciones del sidebar según el rol
* Mostrar mensajes de “No autorizado”
